## Code of Conduct

This project and everyone participating in it is governed by our commitment to creating a welcoming and inclusive environment. By participating, you agree to:

- Be respectful and constructive in all interactions
- Focus on what is best for the OSINT community
- Show empathy towards other community members
- Accept constructive criticism gracefully
- Use welcoming and inclusive language

## How to Contribute

There are several ways you can contribute to OSINT-Kit:

1. **Add new OSINT tools** to the appropriate categories
2. **Update existing tool information** (URLs, descriptions, etc.)
3. **Fix broken links** or outdated information
4. **Improve documentation** and organization
5. **Report issues** with tools or the repository
6. **Suggest new categories** for better organization

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/OSINT-Kit.git
   cd OSINT-Kit
   ```
3. **Create a new branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make your changes** following the guidelines below
5. **Test your changes** to ensure they work correctly
6. **Commit your changes** with a clear commit message
7. **Push to your fork** and create a pull request

## Contribution Guidelines

### Tool Submission Guidelines

When adding new tools, please ensure they meet these criteria:

- **Relevance**: The tool must be directly related to OSINT (Open Source Intelligence)
- **Availability**: The tool should be publicly accessible (free or with free tier)
- **Reliability**: The tool should be actively maintained and functional
- **Legality**: The tool must be legal to use and not promote illegal activities
- **Quality**: The tool should provide genuine value to OSINT practitioners

### Formatting Standards

- **Tool Name**: Use the official name of the tool
- **Description**: Provide a brief, clear description of what the tool does
- **URL**: Include the direct link to the tool
- **Category**: Place tools in the most appropriate existing category
- **Consistency**: Follow the existing format and style

### Example Format

```markdown
- [Tool Name](https://tool-url.com/) - Brief description of what the tool does and its main purpose.
```

### Categories

Tools should be placed in the most appropriate category:

- **OSINT Aggregator Tools**: Comprehensive OSINT platforms
- **Location Tracking**: Tools for tracking and analyzing location data
- **Marketplaces**: Platforms for searching classified ads and marketplaces
- **Miscellaneous**: General-purpose OSINT tools that don't fit other categories
- **Operating Systems**: Security-focused operating systems
- **PGP Key Servers**: Public key infrastructure tools
- **Recording & Auditing**: Tools for documentation and evidence collection

## Pull Request Process

### Before Submitting

1. **Check existing issues** to avoid duplicates
2. **Test your changes** thoroughly
3. **Ensure consistency** with existing formatting
4. **Verify all links** are working and accessible
5. **Update documentation** if needed

### Pull Request Requirements

- **Clear title**: Describe what your PR does
- **Detailed description**: Explain the changes and their purpose
- **Reference issues**: Link to any related issues
- **Screenshots**: Include screenshots for significant changes
- **Testing**: Confirm that all links work and formatting is correct

### Pull Request Template

```markdown
## Description
Brief description of the changes made.

## Type of Change
- [ ] New tool addition
- [ ] Documentation update
- [ ] Link fix
- [ ] Structural improvement
- [ ] Other (please describe)

## Testing
- [ ] All links are working
- [ ] Formatting is consistent
- [ ] No broken markdown
- [ ] Changes follow guidelines

## Additional Notes
Any additional information or context.
```

## Issue Reporting

When reporting issues, please include:

- **Clear title**: Brief description of the issue
- **Detailed description**: What you expected vs. what happened
- **Steps to reproduce**: How to recreate the issue
- **Environment**: Browser, operating system, etc.
- **Screenshots**: If applicable, include screenshots

### Issue Types

- **Bug Report**: Something isn't working as expected
- **Feature Request**: Suggest a new feature or improvement
- **Documentation**: Issues with documentation clarity or accuracy
- **Tool Suggestion**: Propose adding new tools

## Review Process

All contributions will be reviewed by maintainers. The review process includes:

1. **Automated checks**: Link validation and formatting
2. **Manual review**: Content quality and relevance
3. **Community feedback**: Input from other contributors
4. **Final approval**: Maintainer approval for merge

### Review Criteria

- **Relevance**: Does the contribution align with project goals?
- **Quality**: Is the information accurate and well-formatted?
- **Completeness**: Are all required elements included?
- **Consistency**: Does it follow existing patterns and style?
